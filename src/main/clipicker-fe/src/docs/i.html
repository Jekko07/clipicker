CliPicker app

Stack:

Setup

Install the following:

  • JDK1.8
  • Maven
  • MySQL
  • nodejs
  • vue-cli
  • quasar-cli
  • Android SDK
  • cordova
  • STS/Eclipse/Intellij for backend IDE
  • VSCode

For Local Development:

Project Structure
clipicker:
  • Frontend sourcecode
    • src/main/clipicker-fe
  • Backend sourcecode
    • src/main/java
    • src/main/resources
Project Local setup:
  • Install MySQL
    • Create super root user with following credentials
      • username: root
      • password: root
    • Create empty database with below name
      • name: clipicker
  • Running the frontend and backend app
    • Open the clipicker project in IDE
    • Running the frontend clipicker vue app
      • goto the clipicker-fe directory
      • and hit the below command:
        • yarn dev
      • this will open a browser automatically with the url http://localhost:1818
    • Running the backend API service app
      • open the ClipickerApplication.java inside the IDE and click Run or execute it
      • upon running ClipickerApplication it will migrate the schema tables
      • after the backend application has started the port http://localhost:2018 will be serve
      • do some test by going to the URL http://localhost:2018/api/test if it display "CliPicker backend API has successfully started!"
    • Use webserver to serve this 2 running apps (CliPicker Frontend and CliPicker Backend API)
      • Since both apps are already running we need to have same DNS in order to prevent several issues like CSRF and CORS when communicating with each other. Hence, both of them need to be in same domain name (DNS).
      • DNS for local development like localhost we use clipicker as server_name --> this will serve under http://clipicker
      • Append new dns entry in the hosts file as Administrator with below content
        • 127.0.0.1 clipicker
      • For Windows OS the file was located under -> C:\Windows\System32\drivers\etc\hosts
      • For linux OS the file was located under -> /etc/hosts
      • This should look like this
      • After the DNS was created we can now use this as our server name in our local development purposes
      • For Windows we will use NGINX proxy webserver -> it is used for serving static files and reverse proxy app services
      • Download NGINX -> for Windows -> http://nginx.org/download/nginx-1.15.2.zip
      • For Linux OS -> you just have to install via -> apt-get package manager -> sudo apt-get install nginx or sudo yum install nginx
      • After downloading nginx add new server entry inside the file nginx.conf :
        • File location -> /conf/nginx.conf
        • with below content:
        • server {
  listen       80;
  server_name  clipicker;

  location / {
    proxy_pass http://127.0.0.1:1818;
    proxy_buffering off;
  }

  location /api/ {
    proxy_pass http://localhost:2018/;
    proxy_buffering off;
  }

}
        • Start the nginx by running nginx.exe for windows
        • Start the nginx by running sudo service nginx reload for Linux OS
        • Then thats't it
        • The CliPicker App is available now via http://clipicker
